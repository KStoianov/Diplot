Diplot Travel Agency: AI-Intelligent Booking System
Diplot Travel е високотехнологична Full-stack платформа (MERN), която предефинира начина, по който потребителите намират своите почивки. Системата използва локален изкуствен интелект (LLM) за семантичен анализ на потребителските желания, превръщайки обикновения чат в мощна, персонализирана търсачка.

🛠 1. Системни изисквания
За да работи платформата стабилно (особено AI модулът), машината трябва да покрива следните изисквания:

- Софтуерна среда
Node.js: Версия 18.x или по-нова.

MongoDB: Локален сървър, работещ на порт 27017.

Ollama: Задължителен софтуер за хостване на локалния модел. Изтеглете от тук.

- Хардуер (Оптимизирано за RTX 4060)
GPU: NVIDIA GeForce RTX (препоръчително 8GB VRAM).

RAM: Минимум 16GB системна памет.

VRAM Оптимизация: Приложението е конфигурирано с контекстен прозорец num_ctx: 1024, което позволява бърза обработка при 8GB видео памет, без да препълва ресурсите на лаптопа.

2. Инсталация и Първоначална настройка
Следвайте тези стъпки точно, за да подготвите средата за работа:

Стъпка 1: Подготовка на AI Модела
След като инсталирате Ollama, отворете терминал и изтеглете модела:

Bash
ollama pull gemma2:9b
Забележка: Уверете се, че иконата на Ollama е активна в системния трей (tray) преди да стартирате сървъра.

Стъпка 2: Инсталиране на зависимостите
Проектът е разделен на две части – клиент и сървър. Трябва да инсталирате пакетите и за двете:

За Сървъра (Backend):

Bash
cd server
npm install
За Клиента (Frontend):

Bash
cd ../client
npm install
Стъпка 3: Зареждане на базата данни (Seed)
За да работи търсачката, трябва да заредите хотелите в MongoDB. Изпълнете скрипта в папката на сървъра:

Bash
cd ../server
node seed_full.js
 3. Архитектура на "AI Brain"
Проектът използва Хибридна логика на търсене (Hybrid AI-Search). Това не е просто обикновен чат, а интелигентен алгоритъм:

Intent Classification (AI): Моделът анализира съобщението и го напасва към една от 16-те официални категории (напр. „Арктическо преживяване“).

Semantic Bridge (Семантичен мост): Ако няма точни съвпадения, бекендът използва вграден речник, за да търси ключови думи в описанията (напр. „студ“ -> претърсва за хотели с „лед“ и „сняг“).

Hierarchical Fallback: Системата гарантира 3 резултата, като автоматично разширява обхвата (Локация -> Категория -> Глобален рейтинг).

 4. Стартиране на приложението
За да работи проектът, трябва да имате два активни терминала:

Терминал 1: Бекенд сървър
Влезте в папка server и стартирайте:

Bash
node server
Сървърът ще слуша на: http://localhost:5000

Терминал 2: Фронтенд клиент
Влезте в папка client и стартирайте:

Bash
npm start
Приложението ще се отвори на: http://localhost:3000

 5. Отстраняване на проблеми (Troubleshooting)
Грешка $regex has to be a string: Системата е защитена с функцията safeRegex, която предотвратява този краш при неопределени параметри от AI.

AI отговаря бавно: * Проверете дали Windows използва NVIDIA картата за Ollama: Settings -> Graphics -> High Performance.

Затворете тежки приложения (Chrome, игри), които заемат VRAM.

Липсващи хотели: Уверете се, че сте изпълнили node seed_full.js, за да напълните базата данни.

6. Структура на папките

server/ — Backend (Node.js, Express, MongoDB).

server/seed_full.js — Демо данни
